## 1. 프로젝트 개요

### 한 줄 정의

> **Claude를 구독 중인 한국 대학생이 `npx uni-agent install` 한 줄로 설치하는 학업 자동화 MCP 플러그인**

### 핵심 전제

- **대상**: Claude Desktop을 사용하는 한국 대학생
- **진입 장벽**: README를 보고 명령어 하나를 입력할 수 있는 수준이면 충분
- **추가 비용**: 없음 — Claude 구독료에 포함
- **배포**: npm 패키지 (`npx uni-agent install`) + GitHub 오픈소스

### 사용 시나리오

```
사용자: "AI 윤리 기말 발표 준비해줘"
          ↓
Claude Desktop AI: plan_task("AI 윤리 발표") 호출
          ↓
[MCP Pre-Hook] 목표 상태 기록 / 컨텍스트(CLAUDE.md) 자동 주입
          ↓
[MCP Role Router] plan_task → Architect 페르소나 적용
          ↓
도구 실행 → 에러 발생 시 [MCP Post-Hook]이 분석·복구 신호 주입
          ↓
Claude Desktop AI: search_papers → register_references → save_output(ppt)
          ↓
[MCP Post-Hook] 완료 마일스톤 저장, 다음 목표 제안 메타데이터 첨부
```

## 2. 배경 및 문제 정의

### 한국 대학생의 반복 작업 현실

한국 대학생이 매 학기 반복하는 작업:
PPT 제작, 보고서 작성, 자료조사(논문 포함), 노트 정리

### 기존 도구의 한계

| 도구 | 월 비용 | 핵심 한계 |
|------|---------|---------|
| Notion AI | $20+ | HWP 미지원, 한국 학술 양식 없음 |
| Gamma | $8~100 | 한국 대학 발표 양식 없음, PPTX 포맷 깨짐 |
| ChatGPT Plus | $20 | 한국어 학술 특화 파이프라인 없음 |
| 뤼튼 | 무료 | 커스터마이징 불가, 폐쇄적 |

### 이 프로젝트가 채우는 공백

- **에이전트 방식**: 기존 도구들은 "도구 모음"이지, 스스로 판단하는 에이전트가 아님
- **무료 학술 API 통합**: OpenAlex, CrossRef, Semantic Scholar 직접 연동
- **오픈소스**: 커뮤니티가 한국 대학별 양식·템플릿을 직접 추가할 수 있는 구조
- **비용**: Claude 구독 외 추가 비용 없음

## 3. 핵심 설계 철학

### Fat MCP 패턴 (Proxy-Hook 방식)

MCP 서버가 단순 I/O 중계에서 벗어나, **지능형 중간 레이어**로 작동한다.
LLM이 도구를 호출할 때 MCP 서버가 중간에서 가로채고, 검증하고, 상태를 관리하며, 응답에 컨텍스트를 주입한다.

```
Thin MCP (이전):
  LLM 요청 → 도구 실행 → 결과 반환

Fat MCP (현재):
  LLM 요청 → [Pre-Hook: 검증·컨텍스트 주입] → 도구 실행
           → [Post-Hook: 에러 복구·상태 갱신·메타데이터 주입] → 결과 반환
```

> **MCP Sampling 없이 구현** — 이전 Fat MCP 시도가 실패한 이유는 `server.createMessage()`(MCP Sampling)를 사용해 서버에서 LLM을 직접 호출하려 했기 때문(-32601). 현재 방식은 LLM 호출 없이, **도구 응답값을 지능적으로 조작**하는 것으로 동일한 효과를 낸다.

### 5단계 설계 원칙

**원칙 1: Proxy / Hook 시스템**
모든 Tool Call을 래퍼(Wrapper) 함수가 감싼다.
- **Pre-Hook**: 인자 유효성 검사, 보안 위험 명령 차단(ssrf-guard 등), 목표 컨텍스트 자동 주입
- **Post-Hook**: 실행 결과 분석, 에러 발생 시 복구 신호 생성, 상태 저장

**원칙 2: 자율 복구 루프 (Autopilot)**
Post-Hook에서 에러(isError: true)가 감지되면, 에러를 그대로 LLM에게 던지지 않는다.
MCP 서버가 **"이러한 에러가 발생했다. 계획을 수정하고 다시 시도하라"**는 복구 신호를 응답에 첨부한다.
`max_retries` 카운터(기본 3회)를 서버 내부에 유지하여 무한 루프를 방지하고, 초과 시에만 사용자에게 에러를 노출한다.

**원칙 3: 영구 상태 관리 (Stateful Memory)**
MCP 서버가 `~/.uni-agent/state.json`에 현재 목표, 완료된 마일스톤, 에러 이력을 기록한다.
LLM이 컨텍스트를 잃은 것 같을 때 Post-Hook이 **"현재 목표는 [X], 다음 단계는 [Y]"**를 응답 메타데이터에 주입한다.

**원칙 4: 동적 컨텍스트 주입 (Context Engineering)**
MCP 서버 시작 시 작업 디렉토리의 `CLAUDE.md`, `README.md` 등 프로젝트 규칙을 자동으로 읽는다.
Pre-Hook이 도구 응답에 해당 규칙을 은밀하게 첨부해, LLM이 컨벤션을 항상 인지하도록 만든다.

**원칙 5: 다중 역할 라우팅 (Role Router)**
도구 종류에 따라 다른 페르소나(Persona)와 권한 세트를 적용한다.

| 도구 | 역할 | 허용 권한 |
|------|------|---------|
| `plan_task` | Architect | 목표 설정, 구조 설계만 |
| `search_papers` | Researcher | 검색·등록만 |
| `save_output` | Worker | 파일 생성만 |
| `parse_file` | Reader | 읽기 전용 |

### Claude Desktop에 노출되는 인터페이스

기존 5개 I/O 도구에 고수준 진입 도구가 추가된다.

| 도구 | 역할 | 적용 페르소나 |
|------|------|-------------|
| `plan_task` *(신규)* | 목표 설정, 마일스톤 수립, 상태 초기화 | Architect |
| `search_papers` | 학술 논문 검색 (SS + OpenAlex + CrossRef) | Researcher |
| `register_references` | 선택한 논문을 레퍼런스 DB에 등록 | Researcher |
| `save_output` | 마크다운을 PDF/DOCX/MD로 변환·저장 | Worker |
| `list_references` | 저장된 레퍼런스 라이브러리 조회 | Reader |
| `parse_file` | 첨부파일 텍스트 추출 | Reader |

## 5. MCP 도구 설계

### Hook이 감싼 도구 호출 흐름

모든 도구 호출은 동일한 래퍼를 통과한다.

```
Claude AI: plan_task({ goal: "AI 윤리 발표 12장" }) 호출
  │
  ▼ [Pre-Hook]
  ├─ 인자 유효성 검사
  ├─ CLAUDE.md / README.md 읽어 시스템 컨텍스트 로드
  ├─ state.json에 목표 기록 { goal, startedAt, status: 'in_progress' }
  └─ Role Router → Architect 페르소나 주입
  │
  ▼ [도구 실행]
  └─ 마일스톤 계획 생성, 각 단계별 도구 순서 정의
  │
  ▼ [Post-Hook]
  ├─ 에러 없음 → 응답에 "다음 단계: search_papers" 메타데이터 첨부
  ├─ 에러 있음 → retry_count 증가, 복구 힌트 주입
  └─ retry >= max_retries → 사용자에게 에러 노출
```

### 자율 복구 루프 상세

```typescript
// Post-Hook 의사코드
if (result.isError) {
  state.retryCount++;
  if (state.retryCount >= MAX_RETRIES) {
    return { ...result, text: `❌ ${MAX_RETRIES}회 시도 후 실패: ${result.text}` };
  }
  // 에러를 숨기고 복구 신호 주입
  return {
    content: [{
      type: 'text',
      text: `[복구 신호] 다음 에러가 발생했습니다: ${result.text}\n`
          + `계획을 수정하고 다시 시도하세요. (시도 ${state.retryCount}/${MAX_RETRIES})`
    }]
  };
}
state.retryCount = 0;  // 성공 시 리셋
```

### 일반적인 호출 순서 (PPT 생성, Fat MCP 기준)

```
1. plan_task(goal="AI 윤리 발표")
   [Pre-Hook] 목표·컨텍스트 주입 → [Post-Hook] 마일스톤 저장
   → "다음 단계: search_papers" 메타데이터 반환

2. search_papers(topic="AI 윤리", ...)
   [Pre-Hook] Researcher 페르소나 → [Post-Hook] 결과 수 기록
   → PaperResult[] 반환

3. register_references([...선별 논문...])
   → ref_001 ~ ref_006 등록, [Post-Hook] 마일스톤 갱신

4. save_output(content="...", output_type='ppt', title='AI 윤리 발표')
   [Pre-Hook] Worker 페르소나 · 파일명 안전성 검사
   [Post-Hook] 완료 상태 저장, 다음 제안 주입
   → ~/Desktop/AI_윤리_발표_2026-03-05.pdf
```

## 6. 참고자료 입력 및 관리

### 3가지 입력 방식

**방식 1: 검색 결과 등록**
`search_papers` → Claude AI가 선별 → `register_references`.

**방식 2: 파일 직접 파싱**
```
parse_file → 텍스트 추출 → Claude AI가 내용 파악
```

| 파일 형식 | 파싱 방법 |
|----------|---------|
| PDF | pdf-parse → 텍스트 + 메타데이터 추출 |
| PPTX | pptx2json → 슬라이드별 텍스트 + 발표자 노트 |
| DOCX | mammoth → HTML 경유 텍스트 |
| HWP / HWPX | hwp.js (동적 import), 실패 시 graceful fallback |

**방식 3: URL 또는 DOI 제공**
코드에서 `referenceStore.add({ type: 'doi', doi: '...' })`로 직접 등록 가능.

### 레퍼런스 저장 구조

```
~/.uni-agent/
├── references/
│   ├── index.json       ← 전체 레퍼런스 메타데이터 인덱스
│   ├── chunks/          ← Vector DB에 저장된 청크 파일들
│   └── originals/       ← 원본 파일 복사본
└── sessions/
    └── {sessionId}.json ← 세션별 작업 상태
```

### 인용 추적 (Citation Graph)

모든 레퍼런스는 **어디에 사용됐는지** 추적된다.

```
ref_001 (Vaswani 2017)
  └─ citationKey: vaswani2017
  └─ APA: (Vaswani et al., 2017)
```

최종 결과물에 참고문헌이 자동 삽입된다.

## 7. 컨텍스트 관리 및 최적화

### 임베딩 모델

| 항목 | 값 |
|------|-----|
| 모델 | `@xenova/transformers` `multilingual-e5-base` |
| 차원 | 768차원 (고정) |
| 비용 | 무료 (로컬 실행) |
| 첫 실행 | ~280MB 다운로드 (이후 캐시) |
| 실패 시 | TF-IDF fallback (768차원 유지) |

### 문서 청킹 전략

| 문서 유형 | 청킹 방식 |
|---------|---------|
| 학술 논문 | Abstract는 단일 청크(분할 금지), 본문은 단락 기준 (512 토큰, 50 토큰 오버랩) |
| 강의 PPT | 슬라이드 1장 = 청크 1개 (발표자 노트 포함) |
| 보고서 | 섹션 제목 기준 분할, 초과 시 단락 기준 세분화 |

### 캐싱

| 캐시 대상 | TTL | 비고 |
|-----------|-----|------|
| 논문 임베딩 | 영구 | 동일 논문 재사용 시 재계산 없음 (`~/.uni-agent/embedding-cache.json`) |
| 검색 결과 | 1시간 | 동일 검색어 재사용 (searchCache 인메모리) |
| 파싱 결과 | 세션 내 영구 | 동일 첨부파일 재분석 없음 |

### 세션 간 컨텍스트 유지

세션이 끝나도 다음에 이어받을 수 있는 정보를 저장한다.

```json
{
  "userProfile": {
    "university": "한양대",
    "major": "컴퓨터공학",
    "preferredLanguage": "ko"
  },
  "referenceLibrary": ["ref_001", "ref_002"],
  "taskHistory": [
```

## 8. 4대 핵심 기능

### 기능 1: PPT 자동 생성

**사용자 입력**: "딥러닝 기말 발표 15장 만들어줘"
**Claude AI 처리**:
1. 주제 분석 + 슬라이드 구성 설계
2. `search_papers`로 관련 논문 수집
3. `register_references`로 논문 등록
4. Marp 마크다운 직접 작성
5. `save_output`으로 PDF 생성

**출력 형식**: PDF (인쇄·제출용 메인)
**내장 템플릿**: 학부 발표 기본 / 팀 프로젝트 / 세미나 / 미니멀

### 기능 2: 자료조사 + 논문 검색

**지원 데이터 소스**

| 소스 | 대상 | 비용 |
|------|------|------|
| Semantic Scholar | 글로벌 논문 2억+ | 무료 |
| OpenAlex | 글로벌 + 한국어 필터 | 무료 |
| CrossRef | DOI 기반 학술 메타데이터 | 무료 |

**처리 방식**: 3-way 병렬 검색 → 중복 제거 → Claude AI가 관련성 직접 판단 → 선별 → 등록.

### 기능 3: 노트 정리

**사용자 입력**: 강의 PPT, PDF, 날것의 필기 텍스트
**Claude AI 처리**: `parse_file`로 내용 파악 → 구조 파악 → 핵심 개념 추출 → 계층 구조로 재편성
**출력 유형**: 구조화된 마크다운 노트

### 기능 4: 보고서 작성

**사용자 입력**: "탄소중립 정책에 대한 5페이지 보고서 써줘"
**Claude AI 처리**: 목차 설계 → 논문 수집 → 섹션별 작성 → 참고문헌 삽입
**지원 문체**: 경어체(-습니다) / 논문체(-이다) 선택
**출력 형식**: DOCX (Pandoc) 또는 MD (fallback)

### Phase 5 — Fat MCP 리팩토링 (진행 중)

Thin MCP의 한계(에러 시 LLM 당황, 컨텍스트 망실, 단발성 도구 실행)를 극복하기 위해
MCP 서버에 지능형 중간 레이어를 추가한다.

#### 5-1단계: Proxy / Hook 시스템 도입

| 작업 | 파일 | 내용 |
|------|------|------|
| ToolWrapper 구현 | `src/mcp/hook.ts` | Pre/Post Hook을 모든 도구에 적용하는 래퍼 |
| Pre-Hook | `src/mcp/hooks/pre.ts` | 인자 검증, 보안 검사(SSRF 등), 컨텍스트 로드 |
| Post-Hook | `src/mcp/hooks/post.ts` | 에러 분석, 상태 갱신, 메타데이터 주입 |
| server.ts 교체 | `src/mcp/server.ts` | 모든 tool() 핸들러를 ToolWrapper로 감쌈 |

#### 5-2단계: 자율 복구 루프 (Autopilot)

| 작업 | 파일 | 내용 |
|------|------|------|
| ErrorRecovery | `src/mcp/recovery.ts` | Post-Hook 에러 감지 → 복구 신호 생성 |
| RetryState | `src/mcp/state.ts` 일부 | retry_count, max_retries 관리 |
| 복구 메시지 형식 | — | LLM이 재시도하도록 유도하는 표준 응답 형식 정의 |

#### 5-3단계: 영구 상태 관리 (Stateful Memory)

| 작업 | 파일 | 내용 |
|------|------|------|
| StateManager | `src/mcp/state.ts` | 목표, 마일스톤, 에러 이력 JSON 영속화 |
| `plan_task` 도구 | `src/mcp/server.ts` | 새 도구: 목표 설정 + 마일스톤 계획 수립 |
| 메타데이터 주입 | Post-Hook 연동 | 응답 끝에 "현재 목표·다음 단계" 자동 첨부 |

#### 5-4단계: 동적 컨텍스트 주입 (Context Engineering)

| 작업 | 파일 | 내용 |
|------|------|------|
| ContextInjector | `src/mcp/context-injector.ts` | 서버 시작 시 CLAUDE.md·README.md 로드 |
| Pre-Hook 연동 | `src/mcp/hooks/pre.ts` | 도구 호출마다 관련 컨텍스트 슬라이스 주입 |
| 규칙 파싱 | — | `.cursorrules`, `CLAUDE.md` 파싱 후 캐시 |

#### 5-5단계: 다중 역할 라우팅 (Role Router)

| 작업 | 파일 | 내용 |
|------|------|------|
| RoleRouter | `src/mcp/role-router.ts` | 도구 → 페르소나 매핑, 권한 세트 적용 |
| Persona 정의 | `src/mcp/personas.ts` | Architect / Researcher / Worker / Reader |
| 권한 제어 | Role별 허용 도구 목록 | 페르소나 외 도구 호출 시 Pre-Hook에서 차단 |

**Phase 5 산출물**: Fat MCP v3.0.0 · Hook 레이어 · Autopilot · State Manager · Context Injector · Role Router

## 13. 의사결정 로그

### MCP Sampling 시도 및 Thin MCP 전환 (Phase 4)

**배경**:
Phase 4에서 내부 LLM 호출을 `@anthropic-ai/sdk` 직접 호출에서 MCP Sampling(`server.createMessage()`)으로 전환했다. 그러나 실제 Claude Desktop 환경에서 `-32601 Method not found` 오류가 발생하며 모든 에이전트의 LLM 호출이 실패했다.

**결정**: Fat MCP에서 Thin MCP 패턴으로 전환.

```
Fat MCP (실패):
  Claude Desktop → run_task → MCP서버가 LLM 호출 → 실패

Thin MCP (현재):
  Claude Desktop AI (자체 추론)
    → search_papers / register_references / save_output / ...
    → MCP서버는 I/O만 실행
```

**트레이드오프**:
- ✅ MCP Sampling 의존성 완전 제거 — 안정적 동작 보장
- ✅ 서버 코드 대폭 단순화 (orchestrator/writer/research 에이전트 제거)
- ✅ Claude Desktop AI가 최신 모델의 추론 능력을 직접 활용
- ⚠️ MCP 서버가 워크플로우를 통제하지 않음 — Claude AI의 도구 선택에 의존
- ⚠️ 체크포인트 / DAG 복구 개념 불필요해짐 (Claude AI가 대화 맥락으로 처리)

**결과**: 5개 I/O 도구, 테스트 57/57 통과, 빌드 성공.

### 외부 검색 API 교체 (Phase 4)

**결정**: RISS/DBpia/Tavily → OpenAlex/CrossRef로 교체, 웹 검색 제거.

| 이전 | 이후 | 이유 |
|------|------|------|
| RISS | OpenAlex (language:ko) | API 키 불필요, 한국어 필터 지원 |
| DBpia | CrossRef | API 키 불필요, DOI 기반 포괄적 |
| Tavily | 제거 | 학술 작업에 웹 검색 불필요 |

### 임베딩 교체 (Phase 4)

**결정**: Voyage API → `@xenova/transformers` (`multilingual-e5-base`, 768차원).

- Voyage API 없이 로컬에서 실제 신경망 임베딩 가능
- 한국어 포함 100개 이상 언어 지원
- 첫 실행 시 모델 다운로드 (~280MB), 이후 캐시
- 기존 TF-IDF 폴백 대비 품질 대폭 향상

### Thin MCP → Fat MCP (Proxy-Hook 방식) 재전환 (Phase 5)

**배경**:
Thin MCP는 MCP Sampling 없이 안정적으로 동작하지만, 에러 자율 복구, 세션 간 상태 유지, 컨텍스트 자동 주입, 역할 분리 같은 고급 기능을 구현할 수 없다. 이를 해결하기 위해 Fat MCP를 **LLM 서버 호출 없이** 재설계한다.

**핵심 아이디어**:
이전 Fat MCP 실패의 원인은 "MCP 서버에서 LLM을 직접 호출"하려 했기 때문이다. 새 방식은 LLM을 호출하는 대신 **도구 응답값을 지능적으로 조작**하여 동일한 효과를 낸다.

```
기존 Fat MCP (실패):  서버 → createMessage() → LLM 호출 → -32601
새 Fat MCP (Phase 5): 서버 → Post-Hook → 응답에 복구 신호 주입 → LLM이 자발적으로 재시도
```

**5단계 구현 계획**:
1. Proxy/Hook 시스템 — 모든 도구를 래퍼로 감쌈
2. 자율 복구 루프 — Post-Hook 에러 감지 + max_retries 제어
3. 영구 상태 관리 — state.json으로 목표·마일스톤 영속화
4. 동적 컨텍스트 주입 — CLAUDE.md 등 자동 로드·주입
5. 다중 역할 라우팅 — 도구별 페르소나·권한 분리

**트레이드오프**:
- ✅ MCP Sampling 불필요 — Claude Desktop과 호환 보장
- ✅ 에러 자율 복구, 상태 관리, 컨텍스트 주입 모두 구현 가능
- ✅ 역할 분리로 컨텍스트 오염(Context Pollution) 방지
- ⚠️ 서버 복잡도 증가 — Hook 레이어 디버깅 난이도 상승
- ⚠️ 상태 관리 버그 시 LLM이 잘못된 상태를 믿을 위험
