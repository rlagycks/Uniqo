## 1. 프로젝트 개요

### 한 줄 정의

> **Claude를 구독 중인 한국 대학생이 `npx uni-agent install` 한 줄로 설치하는 학업 자동화 MCP 플러그인**

### 핵심 전제

- **대상**: Claude Desktop을 사용하는 한국 대학생
- **진입 장벽**: README를 보고 명령어 하나를 입력할 수 있는 수준이면 충분
- **추가 비용**: 없음 — Claude 구독료에 포함
- **배포**: npm 패키지 (`npx uni-agent install`) + GitHub 오픈소스

### 사용 시나리오

```
사용자: "AI 윤리 기말 발표 준비해줘"
          ↓
Claude Desktop AI가 스스로:
  search_papers → 논문 조사
  register_references → 관련 논문 등록
  (마크다운 직접 작성)
  save_output → PDF 출력

사용자: "교수님 강의자료 올릴게, 이거 바탕으로 보고서 써줘"
  [lecture.pdf 경로 제공]
          ↓
Claude Desktop AI가 스스로:
  parse_file → 자료 분석
  search_papers → 관련 논문 추가 검색
  register_references → 논문 등록
  (보고서 마크다운 직접 작성)
  save_output → DOCX 출력
```

## 2. 배경 및 문제 정의

### 한국 대학생의 반복 작업 현실

한국 대학생이 매 학기 반복하는 작업:
PPT 제작, 보고서 작성, 자료조사(논문 포함), 노트 정리

### 기존 도구의 한계

| 도구 | 월 비용 | 핵심 한계 |
|------|---------|---------|
| Notion AI | $20+ | HWP 미지원, 한국 학술 양식 없음 |
| Gamma | $8~100 | 한국 대학 발표 양식 없음, PPTX 포맷 깨짐 |
| ChatGPT Plus | $20 | 한국어 학술 특화 파이프라인 없음 |
| 뤼튼 | 무료 | 커스터마이징 불가, 폐쇄적 |

### 이 프로젝트가 채우는 공백

- **에이전트 방식**: 기존 도구들은 "도구 모음"이지, 스스로 판단하는 에이전트가 아님
- **무료 학술 API 통합**: OpenAlex, CrossRef, Semantic Scholar 직접 연동
- **오픈소스**: 커뮤니티가 한국 대학별 양식·템플릿을 직접 추가할 수 있는 구조
- **비용**: Claude 구독 외 추가 비용 없음

## 3. 핵심 설계 철학

### Thin MCP 패턴

Claude Desktop AI가 직접 판단하고 작성하며, MCP 서버는 I/O 도구만 제공한다.

```
이전 시도 (Fat MCP — 실패):
  사용자 → run_task → MCP서버 내부에서 LLM 호출 → 실패
  이유: Claude Desktop이 MCP Sampling을 지원하지 않음 (-32601)

현재 방식 (Thin MCP):
  사용자 → Claude Desktop AI (직접 판단·작성)
           → search_papers / register_references / save_output / ...
           → MCP서버는 I/O만 실행
```

### 5가지 설계 원칙

**원칙 1: Claude AI가 주도**
Claude Desktop AI가 의도를 파악하고, 어떤 논문을 쓸지, 슬라이드 구성을 어떻게 할지를 직접 판단한다. 사용자는 목적만 말한다.

**원칙 2: MCP 서버는 I/O만**
MCP 서버는 LLM을 호출하지 않는다. 검색, 파일 저장, 레퍼런스 관리 등 I/O 작업만 수행한다.

**원칙 3: 투명한 진행**
Claude Desktop이 작업하는 동안 어떤 도구를 호출하는지 사용자에게 보여진다.
```
🔍 search_papers("AI 윤리") → 23편 발견
📋 register_references → 6편 등록 완료
💾 save_output(ppt) → ~/Desktop/AI_윤리_발표.pdf
```

**원칙 4: 추가 API 키 없음**
모든 검색 API (Semantic Scholar, OpenAlex, CrossRef)와 임베딩 (`@xenova/transformers`)이 무료이며 키가 불필요하다.

**원칙 5: 로컬 우선**
임베딩 모델(~280MB)을 첫 실행 시 다운로드해 로컬에서 실행. 이후 캐시 사용.

## 4. 시스템 아키텍처

### 전체 흐름

```
사용자 (Claude Desktop 대화)
          │ 자연어 입력 + 파일 경로
          ▼
    Claude Desktop AI
          │ 직접 판단·작성·도구 호출
          ▼
  ┌────────────────────────────────────────────────────┐
  │              uni-agent MCP Server                   │
  │                                                     │
  │  search_papers    ← src/tools/search.ts             │
  │  register_refs    ← src/reference/store.ts          │
  │  save_output      ← src/agents/formatter.ts         │
  │  list_references  ← src/reference/store.ts          │
  │  parse_file       ← src/reference/parser.ts         │
  │                                                     │
  │  ┌──────────────────────────────────────────────┐  │
  │  │            Context Layer                      │  │
  │  │  Vector Store (hnswlib-node, 768차원)         │  │
  │  │  Session / Reference Store                    │  │
  │  │  @xenova/transformers 임베딩                  │  │
  │  └──────────────────────────────────────────────┘  │
  └────────────────────────────────────────────────────┘
          │
          ▼
    외부 API (인증 불필요)       파일 출력
    Semantic Scholar             PDF (PPT)
    OpenAlex                     DOCX (보고서)
    CrossRef                     MD (노트)
```

### Claude Desktop에 노출되는 인터페이스

| 도구 | 역할 |
|------|------|
| `search_papers` | 학술 논문 검색 (SS + OpenAlex + CrossRef 병렬) |
| `register_references` | 선택한 논문을 레퍼런스 DB에 등록 |
| `save_output` | 마크다운을 PDF/DOCX/MD로 변환·저장 |
| `list_references` | 저장된 레퍼런스 라이브러리 조회 |
| `parse_file` | 첨부파일 텍스트 추출 |

## 5. MCP 도구 설계

Claude Desktop AI가 직접 호출하는 5개 도구.

### 일반적인 호출 순서 (PPT 생성)

```
1. search_papers(topic="AI 윤리", keywords=["AI ethics", ...])
   → PaperResult[] 반환

2. Claude AI가 결과 검토, 6편 선별

3. register_references([...6편...])
   → ref_001 ~ ref_006, citationKey 수신

4. Claude AI가 Marp 마크다운 직접 작성
   (12장 구성, (김철수, 2023) 형식 인용)

5. save_output(content="...", output_type='ppt', title='AI 윤리 발표')
   → ~/Desktop/AI_윤리_발표_2026-03-05.pdf 생성
```

### 첨부파일 처리

```
parse_file("/path/to/lecture.pdf")
→ { text: "...", metadata: { title, authors, ... } }
→ Claude AI가 내용 직접 파악 후 활용
```

## 6. 참고자료 입력 및 관리

### 3가지 입력 방식

**방식 1: 검색 결과 등록**
`search_papers` → Claude AI가 선별 → `register_references`.

**방식 2: 파일 직접 파싱**
```
parse_file → 텍스트 추출 → Claude AI가 내용 파악
```

| 파일 형식 | 파싱 방법 |
|----------|---------|
| PDF | pdf-parse → 텍스트 + 메타데이터 추출 |
| PPTX | pptx2json → 슬라이드별 텍스트 + 발표자 노트 |
| DOCX | mammoth → HTML 경유 텍스트 |
| HWP / HWPX | hwp.js (동적 import), 실패 시 graceful fallback |

**방식 3: URL 또는 DOI 제공**
코드에서 `referenceStore.add({ type: 'doi', doi: '...' })`로 직접 등록 가능.

### 레퍼런스 저장 구조

```
~/.uni-agent/
├── references/
│   ├── index.json       ← 전체 레퍼런스 메타데이터 인덱스
│   ├── chunks/          ← Vector DB에 저장된 청크 파일들
│   └── originals/       ← 원본 파일 복사본
└── sessions/
    └── {sessionId}.json ← 세션별 작업 상태
```

### 인용 추적 (Citation Graph)

모든 레퍼런스는 **어디에 사용됐는지** 추적된다.

```
ref_001 (Vaswani 2017)
  └─ citationKey: vaswani2017
  └─ APA: (Vaswani et al., 2017)
```

최종 결과물에 참고문헌이 자동 삽입된다.

## 7. 컨텍스트 관리 및 최적화

### 임베딩 모델

| 항목 | 값 |
|------|-----|
| 모델 | `@xenova/transformers` `multilingual-e5-base` |
| 차원 | 768차원 (고정) |
| 비용 | 무료 (로컬 실행) |
| 첫 실행 | ~280MB 다운로드 (이후 캐시) |
| 실패 시 | TF-IDF fallback (768차원 유지) |

### 문서 청킹 전략

| 문서 유형 | 청킹 방식 |
|---------|---------|
| 학술 논문 | Abstract는 단일 청크(분할 금지), 본문은 단락 기준 (512 토큰, 50 토큰 오버랩) |
| 강의 PPT | 슬라이드 1장 = 청크 1개 (발표자 노트 포함) |
| 보고서 | 섹션 제목 기준 분할, 초과 시 단락 기준 세분화 |

### 캐싱

| 캐시 대상 | TTL | 비고 |
|-----------|-----|------|
| 논문 임베딩 | 영구 | 동일 논문 재사용 시 재계산 없음 (`~/.uni-agent/embedding-cache.json`) |
| 검색 결과 | 1시간 | 동일 검색어 재사용 (searchCache 인메모리) |
| 파싱 결과 | 세션 내 영구 | 동일 첨부파일 재분석 없음 |

### 세션 간 컨텍스트 유지

세션이 끝나도 다음에 이어받을 수 있는 정보를 저장한다.

```json
{
  "userProfile": {
    "university": "한양대",
    "major": "컴퓨터공학",
    "preferredLanguage": "ko"
  },
  "referenceLibrary": ["ref_001", "ref_002"],
  "taskHistory": [
    { "intent": "AI 윤리 PPT 12슬라이드", "completedAt": "2026-03-05T10:30:00Z", "status": "done" }
  ]
}
```

## 8. 4대 핵심 기능

### 기능 1: PPT 자동 생성

**사용자 입력**: "딥러닝 기말 발표 15장 만들어줘"
**Claude AI 처리**:
1. 주제 분석 + 슬라이드 구성 설계
2. `search_papers`로 관련 논문 수집
3. `register_references`로 논문 등록
4. Marp 마크다운 직접 작성
5. `save_output`으로 PDF 생성

**출력 형식**: PDF (인쇄·제출용 메인)
**내장 템플릿**: 학부 발표 기본 / 팀 프로젝트 / 세미나 / 미니멀

### 기능 2: 자료조사 + 논문 검색

**지원 데이터 소스**

| 소스 | 대상 | 비용 |
|------|------|------|
| Semantic Scholar | 글로벌 논문 2억+ | 무료 |
| OpenAlex | 글로벌 + 한국어 필터 | 무료 |
| CrossRef | DOI 기반 학술 메타데이터 | 무료 |

**처리 방식**: 3-way 병렬 검색 → 중복 제거 → Claude AI가 관련성 직접 판단 → 선별 → 등록.

### 기능 3: 노트 정리

**사용자 입력**: 강의 PPT, PDF, 날것의 필기 텍스트
**Claude AI 처리**: `parse_file`로 내용 파악 → 구조 파악 → 핵심 개념 추출 → 계층 구조로 재편성
**출력 유형**: 구조화된 마크다운 노트

### 기능 4: 보고서 작성

**사용자 입력**: "탄소중립 정책에 대한 5페이지 보고서 써줘"
**Claude AI 처리**: 목차 설계 → 논문 수집 → 섹션별 작성 → 참고문헌 삽입
**지원 문체**: 경어체(-습니다) / 논문체(-이다) 선택
**출력 형식**: DOCX (Pandoc) 또는 MD (fallback)

## 9. 플러그인 배포 전략

### 사용자 입장에서의 전체 흐름

```bash
# README에 적힌 이 한 줄이 전부
npx uni-agent install

# 내부에서 자동으로 일어나는 일:
# 1. Node.js 패키지 설치
# 2. claude_desktop_config.json에 MCP 서버 자동 등록
# 3. "@xenova/transformers 모델 첫 실행 시 다운로드" 안내
# 4. "Claude Desktop 재시작하세요" 안내

# 이후 Claude Desktop 재시작하면 바로 사용 가능
```

### 설치 마법사 처리 내용

- OS별 config 경로 자동 감지 (macOS / Windows / Linux)
- 기존 config에 병합 (덮어쓰기 금지)
- Pandoc 설치 여부 확인 후 미설치 시 안내
- **API 키 설정 완전 불필요** — LLM은 Claude Desktop AI 자신, 검색·임베딩 모두 무료

### MCP 서버 등록 구조

```json
{
  "mcpServers": {
    "uni-agent": {
      "command": "node",
      "args": ["/path/to/uni-agent/dist/server.cjs"],
      "env": {}
    }
  }
}
```

> **v2.0.0 대비 변경 없음**: `ANTHROPIC_API_KEY`, `TAVILY_API_KEY`, `RISS_API_KEY`,
> `VOYAGE_API_KEY` 모두 제거된 상태 유지. 추가 비용이나 외부 계정 없이 동작.

### npm 패키지 구조

| 패키지명 | 역할 |
|---------|------|
| `uni-agent` | 설치 마법사 CLI (npx 실행 대상) |
| `@uni-agent/core` | 공통 유틸, 타입, Context Manager |
| `@uni-agent/skill-ppt` | PPT 생성 스킬 (독립 설치 가능) |
| `@uni-agent/skill-research` | 자료조사 스킬 (독립 설치 가능) |
| `@uni-agent/skill-notes` | 노트 정리 스킬 (독립 설치 가능) |
| `@uni-agent/skill-report` | 보고서 작성 스킬 (독립 설치 가능) |

## 10. 기술 스택

### 핵심 기술 선택 이유

**언어: TypeScript / Node.js**
- `npx` 배포에 가장 자연스러운 환경
- Marp CLI(Node.js 기반)와 네이티브 통합
- MCP 공식 SDK TypeScript 완성도가 높음

**MCP SDK: `@modelcontextprotocol/sdk`**
- Anthropic 공식 SDK
- Claude Desktop 연동의 표준
- **Thin MCP 패턴**: 서버는 도구 등록만, LLM 호출 없음

**PPT 생성: Marp CLI**
- Markdown → PDF 직접 변환
- Chromium 내장 (별도 설치 불필요)
- Node.js 네이티브

**전체 스택 요약**

| 카테고리 | 선택 | 대안 |
|---------|------|------|
| MCP SDK | @modelcontextprotocol/sdk | Python mcp |
| LLM 호출 | Claude Desktop AI 자신 (Thin MCP) | @anthropic-ai/sdk (API 키 필요) |
| 런타임 | Node.js v20 LTS | Bun |
| 언어 | TypeScript 5.x | JavaScript |
| PPT 렌더링 | @marp-team/marp-cli | Reveal.js |
| 문서 변환 | Pandoc (CLI) | docx npm 패키지 |

## 13. 의사결정 로그

### MCP Sampling 시도 및 Thin MCP 전환 (Phase 4)

**배경**:
Phase 4에서 내부 LLM 호출을 `@anthropic-ai/sdk` 직접 호출에서 MCP Sampling(`server.createMessage()`)으로 전환했다. 그러나 실제 Claude Desktop 환경에서 `-32601 Method not found` 오류가 발생하며 모든 에이전트의 LLM 호출이 실패했다.

**결정**: Fat MCP에서 Thin MCP 패턴으로 전환.

```
Fat MCP (실패):
  Claude Desktop → run_task → MCP서버가 LLM 호출 → 실패

Thin MCP (현재):
  Claude Desktop AI (자체 추론)
    → search_papers / register_references / save_output / ...
    → MCP서버는 I/O만 실행
```

**트레이드오프**:
- ✅ MCP Sampling 의존성 완전 제거 — 안정적 동작 보장
- ✅ 서버 코드 대폭 단순화 (orchestrator/writer/research 에이전트 제거)
- ✅ Claude Desktop AI가 최신 모델의 추론 능력을 직접 활용
- ⚠️ MCP 서버가 워크플로우를 통제하지 않음 — Claude AI의 도구 선택에 의존
- ⚠️ 체크포인트 / DAG 복구 개념 불필요해짐 (Claude AI가 대화 맥락으로 처리)

**결과**: 5개 I/O 도구, 테스트 57/57 통과, 빌드 성공.

### 외부 검색 API 교체 (Phase 4)

**결정**: RISS/DBpia/Tavily → OpenAlex/CrossRef로 교체, 웹 검색 제거.

| 이전 | 이후 | 이유 |
|------|------|------|
| RISS | OpenAlex (language:ko) | API 키 불필요, 한국어 필터 지원 |
| DBpia | CrossRef | API 키 불필요, DOI 기반 포괄적 |
| Tavily | 제거 | 학술 작업에 웹 검색 불필요 |

### 임베딩 교체 (Phase 4)

**결정**: Voyage API → `@xenova/transformers` (`multilingual-e5-base`, 768차원).

- Voyage API 없이 로컬에서 실제 신경망 임베딩 가능
- 한국어 포함 100개 이상 언어 지원
- 첫 실행 시 모델 다운로드 (~280MB), 이후 캐시
- 기존 TF-IDF 폴백 대비 품질 대폭 향상
